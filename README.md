# home-main-archive

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放和展示多个独立制作的 HTML 页面。这些页面通常为静态内容，不依赖后端服务，可直接在浏览器中打开查看。仓库本身不针对任何特定域名或网站，仅作为页面文件的归档与分发点。

## 目录说明

```
home-main-archive/
├── index.html          # 仓库首页 / 页面索引
├── pages/              # 存放主要 HTML 页面文件
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 页面引用的资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本说明文件
```

- `index.html`：仓库的入口页面，通常包含页面列表或导航。
- `pages/`：所有归档的 HTML 页面存放于此目录。
- `assets/`：页面所需的静态资源，按类型分文件夹存放。

## 页面归档说明

- 每个页面均为独立完整文件，可单独打开运行。
- 页面之间无强依赖关系，可自由访问或引用。
- 资源文件建议使用相对路径引用，以保证在本地或部署后均能正常加载。
- 归档的页面可能包含实验性内容、临时工具、或长期维护的静态页面。

## 维护说明

- 新增页面：在 `pages/` 目录下添加新 HTML 文件，并在 `index.html` 中更新索引（如有需要）。
- 更新页面：直接修改对应文件，注意保持资源引用路径正确。
- 删除页面：移除对应文件及索引中的条目。
- 资源管理：新增资源请放入 `assets/` 对应子目录，避免散落在根目录。
